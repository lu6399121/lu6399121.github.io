Activity的生命周期

Posted on 2016-08-02   |  

Activity是Android开发中最常用的组件，深刻理解Activity的生命周期有助于我们开发强大而灵活的应用。

Activity的三种状态

Activity基本上以三种状态存在：
onResume：位于屏幕前台并具有用户焦点。
onPause：部分可见，该Activity被一个透明的Activity或者Dialog覆盖。
onStop：完全不可见，该Activity被另一个Activity完全覆盖。

基本生命周期方法

以下是基本的生命周期方法：
onCreate(): 首次启动Activity时调用，适合做一些初始化的操作。
onRestart(): 在Activity已停止并再次启动调用，始终后接onStart()，该方法里一般不做任何操作。
onStart():在Activity即将可见时调用，即在onResume()之前调用，与onStop()相对应。
onResume():在 Activity 即将开始与用户进行交互之前调用。 此时，Activity 处于 Activity 堆栈的顶层，并具有用户输入焦点。
onPause():当系统即将开始继续另一个 Activity 时调用。 此方法通常用于确认对持久性数据的未保存更改、停止动画以及其他可能消耗 CPU 的内容，诸如此类。 它应该非常迅速地执行所需操作，因为它返回后，下一个 Activity 才能继续执行。
onStop(): Activity 对用户不再可见时调用。如果 Activity 被销毁，或另一个 Activity（一个现有 Activity 或新 Activity）继续执行并将其覆盖，就可能发生这种情况。
如果 Activity 恢复与用户的交互，则后接 onRestart()，如果 Activity 被销毁，则后接 onDestroy()。
onDestroy(): 在 Activity 被销毁前调用。这是 Activity 将收到的最后调用。 当 Activity 结束（有人调用 Activity 上的 finish()），或系统为节省空间而暂时销毁该 Activity 实例时，可能会调用它。 可以通过 isFinishing() 方法区分这两种情形。适合最一些资源回收操作。

完整的生命周期


跟Activity是三种状态类似，Activity的生命周期分三种：
完整的生命周期： onCreate()->onStart()->onResume()->onPause()->onStop()->onDestroy()。
可见生命周期：发生在onStart()和onStop方法之间。
前台生命周期：发生在onResume()和onPause()之间。

保存 Activity 状态

涉及到onSaveInstanceState()和onRestoreInstanceState()两个方法。需要明确的是onSaveInstanceState()调用在onStop()之前，与onPause()没有既定的时序关系。需要明确的是，正常情况情况下系统不会回调onSaveInstanceState()这个方法，它只会出现在Activity被异常终止的情况下。
1、 该Activity可能会被系统内存回收。比如在该Activity上启动新的Activity、按Home键、锁屏等。
2、 资源相关的系统配置发生改变导致Activity被杀死并重新创建。

View的正常保存和恢复的前提是需要设定控件id，保存和恢复的过程是一种典型的委托思想，上层委托下层，父容器委托子容器。

其他

onCreateOptionsMenu()十分常用，它跟onCreate()方法类似，只会在首次启动Activity时调用。需要注意的是，它是调用在onResume()之后，如果在onResume()中直接操作menuItem，可能会引起空指针异常。
